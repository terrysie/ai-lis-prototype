AI-LIS 第一版原型需求

项目定位

这是一个 AI-LIS 实验室信息管理系统原型，面向单实验室 / 单院区，优先覆盖常规检验主流程。

系统目标不是展示 AI 聊天框，而是让 AI 真正嵌入检验科流程，减少重复人工、缩短处理时间、降低漏项和返工。

V1 核心目标

第一版先做静态前端原型，不接入真实医院系统，不使用真实患者信息，不需要后端和数据库。

V1 重点展示三件事：

1. 智能预审 / 自动放行
2. 危急值自动闭环
3. 内部智能助手

AI 使用原则

AI 不替代最终医疗责任人。

AI 主要负责：

* 筛选
* 解释
* 催办
* 排序
* 风险提示

所有 AI 判定必须可解释、可追溯。

第一版页面重点

请优先做以下模块：

1. 审核技师工作台
2. 样本流转概览
3. AI 预审队列
4. AI 判定卡片
5. 危急值中心
6. 内部智能助手
7. 质控 / 管理概览

AI 预审分级

AI 预审结果分为三类：

A 类：自动放行

典型条件：

* 结果在参考范围内
* 无危急值
* 与历史波动合理
* 仪器和质控状态正常
* 样本状态合格

系统动作：

* 自动进入发布前队列
* 按规则直接放行

B 类：快速复核

典型条件：

* 轻度异常
* 历史变化偏大但可解释
* 项目组合无明显冲突

系统动作：

* 进入快速复核队列
* 审核员可以一键确认或转重点复核

C 类：重点复核

典型条件：

* 危急值
* 结果极端偏差
* 项目间逻辑冲突
* 质控或仪器状态异常

系统动作：

* 必须人工重点审核
* 不允许自动放行

AI 判定卡片格式

AI 判断不要显示大段自然语言，必须采用三段式：

1. 结论
2. 命中规则
3. 建议动作

示例：

结论：该结果可进入快速复核。
命中规则：ALT 轻度升高，历史趋势可解释，未触发危急值。
建议动作：审核员一键确认，或转入重点复核。

危急值中心

危急值中心需要展示：

* 待确认
* 已通知
* 超时催办
* 已闭环

危急值闭环应包含：

* 触发时间
* 通知对象
* 确认状态
* 催办记录
* 留痕状态

内部智能助手

内部智能助手面向检验科内部人员，不面向患者。

示例问题：

* 这条结果为什么没有自动放行？
* 今天哪些危急值还没闭环？
* 这份样本为什么被退样？
* 哪台仪器今天最异常？
* 某患者近 7 天结果趋势如何？

第一版只做模拟问答，不接入真实数据。

UI 要求

* 专业、清爽、医疗信息系统风格
* 不要花哨
* 不要做成普通后台模板
* 按岗位任务台设计
* 高频信息尽量一屏展示
* 状态颜色克制清晰
* 支持 iPad 和手机基本查看

技术要求

第一版使用：

* HTML
* CSS
* JavaScript

所有代码可以先写在 index.html 中。

不需要：

* 后端
* 数据库
* 登录系统
* 真实接口
* 真实患者信息
* 真实医疗判断

第一版目标

做出一个可以通过 GitHub Pages 打开的 AI-LIS 第一层原型，用于展示系统方向、核心页面结构和 AI 嵌入流程的方式。
