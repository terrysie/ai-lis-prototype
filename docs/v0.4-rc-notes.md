# TERRY-LIS v0.4 RC Notes

## Release Candidate Summary

TERRY-LIS v0.4 is a release candidate note for the report and system usability phase.

This is a Windows desktop runnable candidate build. It has verified write loops for the main laboratory workflow, report output and publishing, quality control handling, reagent expiry alert handling, infectious positive alert handling, automated smoke verification, and local Windows packaging candidate artifacts.

This is not a formal production medical system. The current scope remains a desktop prototype and validation candidate for workflow, persistence, audit logging, and packaging.

## Known Limitations

- The current version is not a formal medical production system.
- It is not connected to real laboratory instruments.
- It is not connected to real hospital HIS, LIS, PACS, or EMR systems.
- Production-grade user permissions, audit compliance, electronic signatures, report CA signing, and related governance capabilities are not complete.
- Barcode printing and physical printer integration are not complete.
- Windows packaging artifacts have been locally verified, but there is no formal release channel, automated signing, or auto-update mechanism yet.
- SQLite is suitable for the current prototype and single-machine validation. It is not a multi-terminal production database plan.
- Current AI-related capabilities should be treated as workflow assistance only and must not replace human medical judgment.
