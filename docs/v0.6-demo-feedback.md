# TERRY-LIS v0.6 Demo Feedback Template

## 目的

本模板用于在完成 Demo Guide 或约 10 分钟 walkthrough 后，收集首次体验者对系统定位、路线清晰度和 Demo 限制的反馈。它只准备统一的反馈内容，不提供线上提交、账号、分析或持久化能力。

## 收集原则

- 使用团队约定的人工反馈渠道记录反馈，例如人工汇总的讨论记录或评审表。
- 反馈只描述本地 Demo 的体验，不代表医院正式生产系统验收。
- 不记录真实患者信息、真实检验结果、真实医院账号、联系方式或其他敏感信息。
- 记录页面或步骤时，优先使用模块名称和显示状态，不使用数据库 ID、样本号或患者编号。

## 建议反馈字段

| 字段 | 建议记录内容 |
|---|---|
| 体验范围 | Demo Guide、10 分钟 walkthrough，或具体模块名称。 |
| 路线完成情况 | 已完成、部分完成，或未完成；如未完成，说明停在哪一步。 |
| 系统定位理解 | 是否能理解这是本地 LIS 工作流 Demo，以及不可用于真实医院生产环境。 |
| 入口与文档清晰度 | 哪个页面、入口或文档说明清楚或容易迷路。 |
| 示例数据体验 | 选择的记录状态、观察到的结果，以及是否能按路线继续。 |
| 边界理解 | 是否注意到示例数据、本地原型和非生产用途说明。 |
| 建议 | 可复现的改进建议；不要包含敏感或真实医疗数据。 |

## 最小反馈流程

1. 从 Dashboard 完成或尝试完成推荐体验路线。
2. 按上表记录体验范围、完成情况、问题位置和建议。
3. 将问题归类为文案、导航、示例数据、现有功能行为或超出 v0.6 范围的能力。
4. 对于可复现的问题，补充模块名称、显示状态和复现步骤；不要提交真实数据。

## 不包含的内容

- 不新增反馈 API、数据库表、schema、seed 数据、埋点或分析服务。
- 不新增登录、用户身份、真实医院接口或生产环境能力。
- 不把 Demo 反馈视为真实医疗系统验收、临床决策或真实检验报告依据。
