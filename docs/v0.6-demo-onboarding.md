# TERRY-LIS v0.6 Demo Onboarding

## 背景

v0.6 面向首次体验 TERRY-LIS 的试用者。现有 Demo 已经覆盖多个实验室信息系统工作流，但首次打开时需要更清楚地说明系统定位、可体验范围、推荐路径和产品边界。

## 目标

- 让首次体验者快速理解 TERRY-LIS 是本地实验室信息系统工作流 Demo。
- 让体验者知道当前可浏览和操作的模块。
- 提供一条从工作台到敏感操作覆盖视图的推荐体验路径。
- 明确示例数据、本地原型和非生产用途的边界。

## 首次打开体验原则

- 在 Dashboard 首屏提供简明、可扫描的 Demo Guide。
- 用中文说明系统能力，避免将原型描述为正式医院系统。
- 复用现有导航进入体验步骤，不新增业务写入路径。
- 在功能说明旁明确展示 Demo 边界。

## 可关闭 / 可重新打开 Demo Guide

- 首次打开时，如果浏览器 localStorage 中没有关闭记录，Demo Guide 默认完整展示。
- 用户可以通过“收起 Demo Guide”隐藏整个引导区块；七步推荐流程和 Demo 边界不会被删除。
- 隐藏后，Dashboard 保留“重新打开 Demo Guide”入口，可恢复完整引导内容。
- 关闭状态只保存在浏览器 localStorage 的 `terry-lis-demo-guide-hidden` 键中；值为 `true` 时隐藏，移除该键时恢复默认展示。
- localStorage 只保存 onboarding UI 的可见性，不保存任何业务数据、样本数据、患者数据、权限数据、审核数据或报告数据。
- 本 PR 不新增业务能力、业务写入、数据库写入、登录状态或复杂弹窗 onboarding，只优化首次体验的引导展示行为。

## 推荐体验流程导航

建议按 1 → 7 的顺序体验。该路径覆盖从样本进入实验室，到结果审核、报告发布、风险闭环和权限审计的核心 Demo 流程。

如需按现有示例数据完成一条约 10 分钟的完整路线，请参阅 [v0.6 Demo Walkthrough](v0.6-demo-walkthrough.md)。该路线以页面显示的记录状态选择示例，不依赖固定记录 ID。

| 步骤 | 对应 Demo 模块 | 建议观察与体验 |
|---|---|---|
| 1. 系统概览 | Dashboard / 工作台 | 查看样本流转、审核队列和风险摘要，了解 Demo 的核心模块与运行状态。 |
| 2. 样本接收 | 样本签收 | 确认一个待签收样本，并查看拒收与补采任务。 |
| 3. 结果审核 | 结果审核 | 审核或驳回一个结果，观察权限提示和审计留痕。 |
| 4. 报告流程 | 结果审核 | 当前 Demo 没有独立报告页；从结果审核进入，生成、导出并发布 HTML 报告。 |
| 5. 危急值闭环 | 危急值中心 | 完成电话通知、临床确认和闭环完成状态。 |
| 6. 预警处理 | 质控看板 | 当前 Demo 将质控事件、试剂近效期和传染病阳性预警集中在此处，查看处理入口与状态。 |
| 7. 权限与安全 | 系统设置 | 查看 UI gating、action guard 和敏感操作覆盖视图。 |

本 PR 只优化推荐体验流程的导航与说明：所有入口复用现有页面导航，不新增业务能力、写入逻辑、路由或持久化状态。

## 当前 Demo 可体验能力

- 样本接收：签收、拒收和补采任务。
- 结果审核：审核通过、驳回和权限控制提示。
- 报告流程：HTML 报告生成、导出和正式发布。
- 危急值闭环：电话通知、临床确认和闭环完成。
- 质控事件：事件处理和处理记录。
- 试剂近效期预警：预警查看和处理。
- 传染病阳性预警：预警复核、通知和跟进。
- 权限与安全：UI gating、action guard 和敏感操作覆盖视图。

## Demo 边界

- 当前系统为本地 Demo / 原型系统。
- 当前数据为示例数据。
- 当前系统不连接真实医院系统或真实检验仪器。
- 当前系统不处理真实患者隐私数据。
- 当前系统不作为真实诊疗、真实检验报告或医院生产系统使用。

## 本 PR 不包含的内容

- 不新增数据库 schema。
- 不修改 seed 数据。
- 不修改权限核心逻辑。
- 不引入新依赖。
- 不创建 release。
- 不处理真实医院数据。
