# TERRY-LIS v0.6 Demo Onboarding

## 背景

v0.6 面向首次体验 TERRY-LIS 的试用者。现有 Demo 已经覆盖多个实验室信息系统工作流，但首次打开时需要更清楚地说明系统定位、可体验范围、推荐路径和产品边界。

## 目标

- 让首次体验者快速理解 TERRY-LIS 是本地实验室信息系统工作流 Demo。
- 让体验者知道当前可浏览和操作的模块。
- 提供一条从工作台到敏感操作覆盖视图的推荐体验路径。
- 明确示例数据、本地原型和非生产用途的边界。

## 首次打开体验原则

- 在 Dashboard 首屏提供简明、可扫描的 Demo Guide。
- 用中文说明系统能力，避免将原型描述为正式医院系统。
- 复用现有导航进入体验步骤，不新增业务写入路径。
- 在功能说明旁明确展示 Demo 边界。

## 推荐体验流程

1. 从 Dashboard 查看系统概览、样本流转和风险摘要。
2. 进入样本签收，确认一个待签收样本，并查看拒收与补采任务。
3. 进入结果审核，审核或驳回一个结果，查看权限提示和审计留痕。
4. 在结果审核页生成、导出并发布 HTML 报告。
5. 进入危急值中心，完成电话通知与临床确认的闭环流程。
6. 查看质控事件、试剂近效期预警和传染病阳性预警处理。
7. 进入系统设置，查看权限与敏感操作覆盖视图。

## 当前 Demo 可体验能力

- 样本接收：签收、拒收和补采任务。
- 结果审核：审核通过、驳回和权限控制提示。
- 报告流程：HTML 报告生成、导出和正式发布。
- 危急值闭环：电话通知、临床确认和闭环完成。
- 质控事件：事件处理和处理记录。
- 试剂近效期预警：预警查看和处理。
- 传染病阳性预警：预警复核、通知和跟进。
- 权限与安全：UI gating、action guard 和敏感操作覆盖视图。

## Demo 边界

- 当前系统为本地 Demo / 原型系统。
- 当前数据为示例数据。
- 当前系统不连接真实医院系统或真实检验仪器。
- 当前系统不处理真实患者隐私数据。
- 当前系统不作为真实诊疗、真实检验报告或医院生产系统使用。

## 本 PR 不包含的内容

- 不新增数据库 schema。
- 不修改 seed 数据。
- 不修改权限核心逻辑。
- 不引入新依赖。
- 不创建 release。
- 不处理真实医院数据。
