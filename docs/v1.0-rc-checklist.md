# TERRY-LIS v1.0 RC Checklist

This checklist is the release-control baseline for a Windows-local TERRY-LIS
Demo. It is not evidence that a release candidate has passed. Record the
candidate commit, command output, artifact hashes, and manual-test evidence
for the specific RC being evaluated.

## Candidate Record

Record these values before execution:

| Field | Record |
| --- | --- |
| Candidate branch | `<branch>` |
| Candidate commit | `<full commit SHA>` |
| `origin/main` commit at start | `<full commit SHA>` |
| Package version | `<package.json version>` |
| Windows architecture | `<x64 or actual architecture>` |
| Operator and date | `<name / date>` |

## Smoke Script Inventory

The current package defines these ten smoke scripts. Run them in the listed
order after `db:reset` and `db:check` unless a failure requires investigation:

```text
npm.cmd run smoke:sample-reception
npm.cmd run smoke:result-review
npm.cmd run smoke:critical-values
npm.cmd run smoke:e2e-workflow
npm.cmd run smoke:report-output
npm.cmd run smoke:report-publish
npm.cmd run smoke:system-rules
npm.cmd run smoke:qc-events
npm.cmd run smoke:reagent-expiry-alerts
npm.cmd run smoke:infectious-alerts
```

## Evidence Location Rules

- Keep command output, generated artifacts, archives, and hashes outside the
  Git commit unless a later release decision explicitly calls for a tracked
  document update.
- Do not commit `dist/`, `data/`, SQLite files, installers, ZIP files, or build
  logs.
- The final GitHub Release body may summarize outcomes only after the evidence
  above has been collected for the actual RC.

## Decision Rule

No-Go is the default when a blocking row is missing, fails, or has no evidence.
This checklist supports a local Demo release only. It does not approve real
hospital integration, real patient data, clinical reporting, or production use.
