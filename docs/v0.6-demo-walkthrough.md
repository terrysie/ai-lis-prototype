# TERRY-LIS v0.6 Demo Walkthrough

## 演示前准备

- 从 Dashboard 打开 Demo Guide；如已收起，可使用“重新打开 Demo Guide”。
- 本路线使用本地 SQLite 示例数据。请按页面显示的状态选择记录，不要依赖样本号、患者编号或内部 ID。
- 如需恢复初始示例数据，请在项目根目录运行 `npm.cmd run db:reset`，再重新打开或刷新当前 Demo 页面。该操作只重置本地 Demo 数据；Demo Guide 的显示偏好仍由浏览器 localStorage 管理。

## 约 10 分钟推荐流程

| 时间 | 入口 | 寻找的示例记录 | 操作 | 预期结果 |
|---|---|---|---|---|
| 约 1 分钟 | Dashboard / 工作台 | 不需要选择记录 | 查看样本流转、审核队列和风险摘要。 | 了解本地 Demo 覆盖的模块和当前模拟运行状态。 |
| 约 1 分钟 | 样本签收 | 状态为“待签收”的样本 | 选择记录并确认签收；查看流转记录。 | 样本状态和本地流转记录更新。 |
| 约 2 分钟 | 结果审核 | 状态为“待审核”的结果 | 查看 AI 建议、规则和详情后执行“审核通过”。 | 审核状态与审核留痕更新；该结果可用于下一步报告流程。 |
| 约 1 分钟 | 结果审核 | 已审核且可生成报告的结果 | 生成 HTML 报告、导出报告并正式发布。 | 看到报告生成、导出和发布的本地 Demo 反馈。 |
| 约 2 分钟 | 危急值中心 | 已通知且详情中仍有待确认通知的危急值 | 查看已有通知记录，完成临床确认和闭环。 | 时间轴、当前状态和本地审计留痕更新。 |
| 约 2 分钟 | 质控看板；AI 预审 | 处理中质控事件、未关闭近效期预警、待处理传染病阳性预警 | 先处理质控事件或试剂近效期预警，再在 AI 预审处理阳性预警。 | 对应预警状态与审计反馈更新。 |
| 约 1 分钟 | 系统设置 | 不需要选择记录 | 查看“敏感操作覆盖清单”。 | 理解 UI gating、action guard、audit log 和 demo_operator 的 Demo 边界。 |

## 示例数据选择原则

- 使用页面上仍可操作的记录，例如“待签收”“待审核”“已通知且待确认”“处理中”或未关闭预警。
- 结果审核后的状态会影响报告按钮；优先承接上一步刚审核通过的结果。
- 危急值、质控、试剂和传染病预警的操作会改变本地示例数据。需要重新演示时，先执行数据 reset。
- 不在文档或页面中固定引用数据库 ID、样本号或患者编号，避免 reset、升级或操作后引用失效。

## 数据 Reset 后如何重新开始

1. 运行 `npm.cmd run db:reset` 恢复项目提供的本地示例数据。
2. 重新打开应用或刷新 Demo 页面，使队列重新读取本地数据。
3. 按本路线从 Dashboard 开始，并按显示的状态重新选择可操作记录。

## 当前 Demo 限制

- 本路线只引导现有页面和已有操作，不新增业务能力、数据库 schema、seed 数据、权限核心逻辑或依赖。
- 页面中的操作、报告和审计记录仅用于本地 Demo 体验；部分原型按钮仍只展示模拟交互。
- Demo Guide 的关闭状态只使用 `terry-lis-demo-guide-hidden` localStorage 键，不保存业务、样本、患者、权限、审核或报告数据。

## 使用边界

- TERRY-LIS 是本地 Demo / 原型系统，当前数据均为示例数据。
- 系统不连接真实医院 LIS、真实检验仪器或生产环境。
- 系统不处理真实患者隐私数据。
- 系统不能作为真实诊疗、真实检验报告或医院生产系统使用。
