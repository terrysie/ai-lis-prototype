# TERRY-LIS v0.4 RC Checklist

## Purpose

This document is the final regression checklist before freezing v0.4. It is a release-control checklist, not a new feature description.

v0.4 remains a release candidate for the report and system usability phase. It is not a formal medical production system.

## Baseline Commit

- Base main commit before this checklist: `daf99fb`
- Related RC notes: `docs/v0.4-rc-notes.md`
- README entry: `README.md` v0.4 Release Candidate section

## Required Git State

Before final verification starts, the following conditions must be true:

- Branch is `main` when final verification starts.
- `git status -sb` is clean and equals `## main...origin/main`.
- `git ls-files --others --exclude-standard` returns no untracked files.
- `build-log.txt` and `build-dir-log.txt` do not exist.
- `dist` may exist locally but must not be committed.
- `data/terry-lis.sqlite` may exist locally but must not be committed.
- `package-lock.json` must not be changed by this checklist step.

## Required Regression Commands

The following commands must be run manually during final verification:

```powershell
npm.cmd run db:reset
npm.cmd run db:check
npm.cmd run smoke:sample-reception
npm.cmd run smoke:result-review
npm.cmd run smoke:critical-values
npm.cmd run smoke:e2e-workflow
npm.cmd run smoke:report-output
npm.cmd run smoke:report-publish
npm.cmd run smoke:system-rules
npm.cmd run smoke:qc-events
npm.cmd run smoke:reagent-expiry-alerts
npm.cmd run smoke:infectious-alerts
```

## Expected Regression Result

- All ten smoke tests should pass.
- `db:reset` and `db:check` should pass.
- No schema, seed, package, database, `dist`, build log, or export artifact changes should be introduced.
- Any failure blocks the v0.4 freeze until investigated.

## Windows Build Candidate Confirmation

Confirm the following after local packaging:

- `dist/win-unpacked/TERRY-LIS.exe` exists locally after packaging.
- `dist/TERRY-LIS-1.0.0-Windows-x64.exe` exists locally after packaging.
- The unpacked EXE opens normally.
- The installer installs and opens normally.
- Pages and navigation work normally.
- `dist` remains a local artifact and is not committed.

## Documentation Confirmation

Confirm the following documentation state:

- `README.md` contains a v0.4 Release Candidate entry.
- `docs/v0.4-rc-notes.md` exists.
- `docs/v0.4-rc-checklist.md` exists.
- RC notes state known limitations clearly.
- The current system is not described as a production-ready medical LIS.

## Freeze Acceptance Criteria

v0.4 can enter freeze candidate status only when all of the following are true:

- `main` is synchronized with `origin/main`.
- All required regression commands pass.
- README and RC notes are present.
- No forbidden files enter Git.
- The Windows build candidate has been manually verified.
- Known limitations are documented.
- No new feature work is added during freeze.

## Explicit Non-Goals

This step does not include:

- No new LIS workflow features.
- No schema changes.
- No seed changes.
- No AI behavior changes.
- No instrument or HIS/LIS integration.
- No release signing or auto-update implementation.
- No production compliance claim.

## Final Note

This checklist is a release-control artifact for v0.4 RC. It does not replace production validation, hospital acceptance testing, regulatory review, or clinical governance.
