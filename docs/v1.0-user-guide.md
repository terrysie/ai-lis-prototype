# TERRY-LIS v1.0 User Guide

## Purpose And Boundaries

TERRY-LIS is a Windows-local LIS workflow Demo that uses example data. It can
be used to demonstrate laboratory workflow screens and controls. It is not
connected to a real hospital or laboratory instrument, must not receive real
patient data, and must not be used for diagnosis, clinical reporting, or
hospital production work.

## Installer Use

The final v1.0 installer name will follow the release asset record, for example
`TERRY-LIS-<version>-Windows-<arch>.exe`. Run the downloaded installer, select
the installation directory if required, then launch TERRY-LIS from its Start
Menu or desktop shortcut. The installer and this launch path must be manually
verified for the specific release before publication.

## Portable Use

The final portable archive will follow
`TERRY-LIS-<version>-Windows-<arch>-portable.zip`. Extract the entire archive
to a writable local folder. Do not run it inside the ZIP file. Start
`TERRY-LIS.exe` from the extracted folder. The portable launch is a release
acceptance check and must be confirmed for the final asset.

## First Open And Recommended Demo

On first launch, read the Demo Guide. It states the local-Demo boundary and can
be hidden or reopened from the existing interface. Use the recommended
seven-step order and the 10-minute walkthrough:

1. Review the Dashboard overview.
2. Open Sample Reception and work with an example sample that is pending a
   reception action.
3. Open Result Review and observe approval or rejection with permission-aware
   actions.
4. Open the report area to generate, export, and publish an example report.
5. Open Critical Values to follow notification, clinical confirmation, and
   closure.
6. Visit quality control, reagent-expiry, and infectious-disease alert areas.
7. Review permissions and sensitive-action coverage views.

For detailed route guidance, see
[the v0.6 Demo walkthrough](v0.6-demo-walkthrough.md).

## Example-Data Reset And Integrity Check

Reset and integrity commands are available in a source checkout, not as a
packaged-Demo reset feature. From the repository root on Windows, run:

```powershell
npm.cmd run db:reset
npm.cmd run db:check
```

`db:reset` recreates the source-checkout example data and `db:check` verifies
its integrity. Running either command changes local data; do not run it against
anything other than the Demo data. The release candidate must document any
packaged reset procedure only after it has been tested.

## Local Data And Uninstall

The command-line default database path is `data/terry-lis.sqlite`; it can be
overridden with `TERRY_LIS_DB_PATH`. The Electron application uses its user-data
location so that application data is not written into the installation folder.
Uninstalling the application is not evidence that all local user data has been
removed. Check the tested release instructions and remove local Demo data only
when that is appropriate for the machine.

## Common Startup Checks

- Extract the complete portable ZIP before starting `TERRY-LIS.exe`.
- Use a writable local folder rather than a protected or network location.
- Confirm that security software has not quarantined the downloaded asset.
- When running from source, install the repository dependencies first and start
  the existing application command with `npm.cmd run start`.
- For a reproducible source Demo state, use the reset and integrity commands
  above before repeating the walkthrough.

## Current Demo Limits

TERRY-LIS demonstrates existing local workflow behavior with example data. It
does not provide real hospital integration, instrument connectivity, patient
privacy handling, clinical validation, production support, or a substitute for
a formal hospital LIS.
