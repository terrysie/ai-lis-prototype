## Positioning

TERRY-LIS v1.0 is a Windows-local laboratory information system workflow Demo
and prototype. It is intended for installation or portable demonstration with
example data. It is not connected to a real hospital system, does not process
real patient information, and must not be used for clinical decisions, real
laboratory reports, or hospital production operations.

## Main Demo Capabilities

- Demo onboarding and a recommended seven-step navigation path.
- A 10-minute walkthrough that uses the available example-data workflow.
- Sample reception, sign-off, rejection, and recollection-task demonstration.
- Result review with approval, rejection, permission gating, and action guards.
- HTML report generation, export, and formal-publication workflow.
- Critical-value phone notification, clinical confirmation, and closure flow.
- Quality-control event handling and records.
- Reagent-expiry and infectious-disease positive alert handling.
- Permission-aware sensitive actions and coverage views.

## Known Limitations

- This is a local Demo and prototype, not a hospital production system.
- The Demo uses example data only; real patient data must never be entered.
- No real hospital LIS, laboratory instrument, or production integration is
  provided.
- The application is not a validated clinical reporting or diagnostic system.
- Packaged-data reset behavior must be described from the tested RC; the source
  checkout reset command is not a substitute for a packaged reset workflow.
